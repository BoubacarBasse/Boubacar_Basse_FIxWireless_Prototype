✓ Already Built Feature exists in RepairQ today

Customer SMS notifications are available under Settings → Notifications in RepairQ. No development work required — this is a configuration task.


Josh Muir (CCO) confirmed setup assistance for Sarah Kim's 3 locations this week.

🚀 Setup — 62 Locations
1
Log into RepairQ for each location
2
Go to Settings → Notifications
3
Enable SMS and configure templates
4
Test with one repair ticket

Estimated time: ~30 min across all 62 locations with a trained ops coordinator.

⚠ Phase 2 Estimated Completion Time

Sarah's request to include "estimated ready by 3pm Thursday" in the SMS requires a new field in the RepairQ data model. The current API has no estimated_completion_at field.


  • Repair type durations exist in the API (estimated_duration_minutes)
  • A calculated estimate can be derived from check-in time + repair type
  • Engineering effort required: new field + notification template variable
📊 Impact Projected Reduction

Dana estimated enabling notifications could reduce "where's my phone" inbound calls by ~20%, freeing counter staff to process more repairs — directly increasing revenue per location.
